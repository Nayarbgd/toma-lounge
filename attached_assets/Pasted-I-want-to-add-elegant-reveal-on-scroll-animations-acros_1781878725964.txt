I want to add elegant "reveal on scroll" animations across the website.

The goal is to make sections appear smoothly as the user scrolls, creating a premium and cinematic experience similar to luxury restaurants, hotels and high-end Dubai websites.

IMPORTANT:
The animations must be subtle, highly optimized and feel equally smooth on desktop and mobile devices.

The website should never feel laggy or heavy while scrolling.

Tech stack:
React + Vite + TypeScript
Tailwind CSS
Framer Motion is already installed.

=================================================
ANIMATION PHILOSOPHY
====================

The experience should feel:

* Premium
* Elegant
* Cinematic
* Calm
* Sophisticated
* Lightweight
* Responsive

Elements should gently reveal themselves as the user scrolls, almost like magic, but without calling attention to the animation itself.

The user should feel smoothness, not effects.

=================================================
MOBILE-FIRST PERFORMANCE REQUIREMENTS
=====================================

Performance is a top priority.

The animations must behave smoothly on:

* Desktop
* Tablets
* iPhone
* Android devices
* Low-powered devices

Use:

* Framer Motion
* Intersection Observer
* whileInView
* viewport={{ once: true, amount: 0.20 }}

Animations should trigger ONLY ONCE.

Do NOT continuously animate while scrolling.

Avoid:

* Continuous scroll listeners
* requestAnimationFrame loops
* heavy parallax
* expensive blur animations
* repeated re-renders
* large transforms
* large shadows during animation
* animation libraries on top of Framer Motion

Respect:

prefers-reduced-motion

If the user has reduced-motion enabled:

* disable translations
* disable scaling
* use simple fade only

=================================================
GPU OPTIMIZATION
================

Animations should use only:

* opacity
* transform
* translateX
* translateY
* scale

Avoid animating:

* width
* height
* top
* left
* box-shadow
* filter
* backdrop-filter

Use:

will-change:
transform, opacity;

where appropriate.

=================================================

<Reveal>

Supported variants:

fadeUp

=================================================
MOBILE RULES
============

On screens below 768px:

Reduce translations:

Desktop:
30px

Mobile:
15px

Reduce duration:

Desktop:
650ms

Mobile:
450ms

Disable stagger delays larger than:

100ms

Never block scrolling.

Animations must never create:

* frame drops
* delayed touch response
* stuttering
* layout shifts

Scrolling should remain at 60fps whenever possible.

=================================================

100ms
