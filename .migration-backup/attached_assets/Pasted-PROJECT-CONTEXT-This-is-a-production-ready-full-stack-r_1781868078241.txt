PROJECT CONTEXT

This is a production-ready full-stack restaurant reservation application called Toma Lounge.

Tech stack:

* Frontend: React 18 + Vite + TypeScript
* UI: Tailwind + shadcn/ui
* State management: TanStack Query
* Router: Wouter
* Backend: Express 5 + TypeScript
* Database: Supabase PostgreSQL
* Auth: Supabase Auth
* Realtime: Supabase Realtime
* Monorepo: pnpm workspaces

IMPORTANT RULES

1. DO NOT refactor unrelated code.
2. DO NOT change the existing reservation API contracts.
3. DO NOT remove or modify existing features.
4. DO NOT rename existing files unless absolutely necessary.
5. DO NOT break current realtime subscriptions.
6. DO NOT change authentication flow.
7. Preserve TypeScript strict mode.
8. Prefer additive changes over destructive changes.
9. Create migrations incrementally.
10. If a schema change is required, explain the migration before applying it.

GOAL

Upgrade the admin dashboard into a more complete restaurant management system while keeping the current reservation flow fully functional.
